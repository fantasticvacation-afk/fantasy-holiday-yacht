## 一、整体网站结构与全局整改指令

### 1. 全局信息架构升级（对标 Benetti / Feadship）

- 以「角色导向」为第一层逻辑，而不是"公司自述"。

  - 「即将成为船东」→ 全案定制 + 全系游艇。
  - 「想体验一次顶级海上假期」→ 租赁航线。
  - 「已有游艇需要专业托管」→ 托管维保。

- 所有主栏目都必须有英文版：

## 五、全系游艇（yachts.html）页面整改指令

- 这个页面要对标 Benetti / Ferretti 的"Fleet / Yachts"分类页。

1. 分类筛选
- 按 3–4 个维度可筛选：
  - 系列：君临 / 远征 / 飞桥 / （其他）。
  - 长度：18–30m / 30–50m / 50m+ / 80m+。
  - 用途：私人生活 / 租赁优先 / 探险型。
  - 动力：传统柴油 / 混合动力 / 电动。

2. 单船详情页（yacht detail）模板
- 每艘船详情页必须包含：
  - 船名 + 亮点一句（如"可携带直升机与潜水艇的 82 米传奇"）。
  - 技术参数表（LOA / Beam / Draft / GT / Engine / Max Speed / Cruise / Range / Guests / Crew / Class）。
  - 平面图区域（至少一张布局图，可用平面示意图）。
  - 3–6 张高清图片（外景 + 内景）。
  - 1–2 行"典型使用场景"说明。
  - 1–2 个 CTA：
    - "索取技术参数与配置单" → contact 表单，自动标记"咨询类型=游艇详情"。
    - "预约看船/试航" → contact。

3. Listing 页 UI
- 每个游艇在列表页中，只显示：船名 + 尺寸 + 简版价格区间（或"价格面议"）+ 1 张图。
- 点击后进入详情页。
